# 快速开始：编译与启动（零代码）

- 目标：不写业务代码，仅通过配置启动完整 API 服务。
- 前提：已在 `configs/` 下准备好 `project.yaml`、`interfaces/` 与可选 `hooks/`。

## 编译
- Windows（在项目根目录执行）：
```
 go build -o bin/server ./cmd/server
```
- 生成的可执行文件：`bin/server`（Windows 为 `bin/server.exe`）。

## 启动
- 在项目根目录运行：
```
 bin/server.exe
```
- 服务默认监听 `:8080`；可通过 `configs/project.yaml` 的 `server.port` 修改。

## 目录约定
- `configs/project.yaml`：项目运行参数（端口、超时、CORS、限流、数据源、插件、认证）
- `configs/interfaces/`：接口定义（方法/路径/认证/限流/事务/文档与步骤 DSL）
- `configs/hooks/`：钩子配置（Auth/Before/After，可按接口匹配与覆盖）
- `storage/`：本地 Blob 存储（上传文件保存目录）

## 热重载（可选）
- 将 `server.autoReload` 设为 `true` 后，服务会自动检测 `configs/` 变更并重建路由。

## 验证
- 打开浏览器或使用 curl 访问配置中的接口路径，如：`GET /docs`、`GET /openapi.json` 或业务接口 `/api/...`
