# 学习路径指南

本文档提供框架的线性学习路径，帮助用户从零开始逐步掌握框架的使用。

## 学习阶段概览

### 第一阶段：基础入门（1-2天）
**目标**：完成环境搭建和第一个接口配置

1. **环境准备与项目启动**
   - 阅读：[01-快速开始-编译与启动.md](./01-快速开始-编译与启动.md)
   - 实践：编译框架并启动服务
   - 验证：访问 `http://localhost:8080/docs` 确认服务正常运行

2. **项目配置理解**
   - 阅读：[02-项目配置.md](./02-项目配置.md)
   - 实践：修改 `project.yaml` 配置数据库连接和基本参数
   - 关键配置项：
     - `server.port`: 服务端口
     - `server.autoReload`: 热重载配置
     - `datasources`: 数据源配置
     - `auth.jwt`: JWT认证配置

3. **创建第一个简单接口**
   - 阅读：[03-接口配置.md](./03-接口配置.md) 中的根级配置部分
   - 实践：创建一个简单的GET接口返回静态数据
   - 示例配置：
     ```yaml
     module: demo
     endpoint: hello
     method: GET
     path: /api/hello
     steps:
       - response:
           status: 200
           body:
             message: "Hello, World!"
     ```

### 第二阶段：核心功能掌握（3-5天）
**目标**：掌握核心步骤DSL和数据处理能力

4. **步骤DSL深入学习**
   - 阅读：[04-步骤DSL-参考.md](./04-步骤DSL-参考.md) 中的响应、校验、映射部分
   - 实践：创建带参数校验和数据转换的接口
   - 关键步骤类型：
     - `response`: 构建HTTP响应
     - `validate`: 参数校验
     - `transform`: 数据映射转换

5. **数据操作能力**
   - 阅读：[04-步骤DSL-参考.md](./04-步骤DSL-参考.md) 中的SQL操作部分
   - 实践：创建CRUD接口操作数据库
   - 关键步骤：
     - `sql.query`: 查询数据
     - `sql.exec`: 执行增删改操作

6. **认证与权限控制**
   - 阅读：[05-认证与授权.md](./05-认证与授权.md)
   - 实践：为接口添加JWT认证和权限控制
   - 关键配置：
     - `auth: jwt` 接口认证方式
     - `roles: [user, admin]` 角色控制
     - `permissions: [user.read]` 权限控制

### 第三阶段：高级功能应用（5-7天）
**目标**：掌握高级特性和插件开发

7. **文件处理能力**
   - 阅读：[07-文件上传与存储.md](./07-文件上传与存储.md)
   - 实践：实现文件上传下载接口
   - 关键步骤：
     - `upload.save`: 文件上传保存
     - 配置数据源支持文件存储

8. **插件集成与开发**
   - 阅读：[10-插件集成指南.md](./10-插件集成指南.md)
   - 阅读：[17-插件能力与编写指南.md](./17-插件能力与编写指南.md)
   - 实践：集成现有插件或开发简单插件

9. **系统运维与监控**
   - 阅读：[12-Admin与运维.md](./12-Admin与运维.md)
   - 实践：使用管理步骤监控系统状态
   - 关键管理步骤：
     - `admin.docs`: 获取接口文档
     - `plugins.status`: 查看插件状态
     - `obs.metrics`: 获取运行指标

### 第四阶段：生产环境部署（2-3天）
**目标**：完成生产环境配置和优化

10. **性能与安全配置**
    - 阅读：[06-限流与防重放.md](./06-限流与防重放.md)
    - 实践：配置接口限流和防重放保护
    - 关键配置：
      - `rateLimit`: 接口限流配置
      - `nonce`: 防重放配置

11. **环境管理与部署**
    - 阅读：[16-环境变量与扩展.md](./16-环境变量与扩展.md)
    - 实践：配置多环境部署和敏感信息管理

12. **文档与API管理**
    - 阅读：[11-文档与OpenAPI-生成与发布.md](./11-文档与OpenAPI-生成与发布.md)
    - 实践：生成并发布API文档

## 常见学习误区

1. **不要一开始就尝试复杂配置**：从简单的响应步骤开始，逐步增加复杂度
2. **重视数据源配置**：正确的数据源配置是后续所有操作的基础
3. **理解上下文变量传递**：步骤之间的数据通过上下文变量传递，这是DSL的核心概念
4. **善用模板函数**：模板函数可以大大简化数据处理逻辑
5. **定期检查配置规范**：使用 `admin.lint` 步骤检查配置规范性

##  troubleshooting 指南

### 服务启动问题
- 检查端口是否被占用
- 验证数据源连接配置
- 检查YAML语法是否正确

### 接口访问问题
- 确认接口路径和方法是否正确
- 检查认证和权限配置
- 查看步骤执行日志

### 数据操作问题
- 验证SQL语句语法
- 检查数据库连接状态
- 确认数据表结构和字段名

### 插件相关问题
- 检查插件二进制文件权限
- 验证插件配置参数
- 查看插件运行日志
